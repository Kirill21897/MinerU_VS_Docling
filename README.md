# PDF Extract Kit: MinerU vs Docling

## Описание проекта
Этот проект представляет собой мощное решение для конвертации PDF-документов в формат Markdown. Он объединяет и позволяет сравнить работу двух передовых инструментов извлечения данных:

1.  **MinerU (PDF-Extract-Kit)**: Высокоточный инструмент на базе моделей глубокого обучения (Deep Learning), специализирующийся на сложной верстке, распознавании формул, таблиц и научных документов.
2.  **Docling**: Современный, быстрый и легковесный конвертер документов, обеспечивающий отличные результаты при минимальных затратах ресурсов.

Проект полностью настроен для локального использования, обеспечивая конфиденциальность данных и удобство работы.

## Особенности
*   **Два режима работы**: Запускайте конвертацию через MinerU, Docling или оба сразу для наглядного сравнения качества.
*   **Автоматическая организация**: Просто поместите файлы в папку `data/raw`, и готовые Markdown-файлы появятся в `data/processed`.
*   **Локальные модели**: Полная автономность благодаря локальному использованию моделей нейросетей (без API-ключей и передачи данных).
*   **Продвинутое извлечение**: Поддержка распознавания таблиц, математических формул и сохранение структуры документа.
*   **Экспорт медиа**: Автоматическое извлечение и сохранение изображений из PDF.

## Установка

### Предварительные требования
*   **Python 3.10** или выше.
*   **CUDA** (рекомендуется): Наличие видеокарты NVIDIA ускорит работу моделей MinerU, но работа на CPU также поддерживается.

### Пошаговая инструкция

1.  **Клонируйте репозиторий:**
    ```bash
    git clone <ссылка-на-ваш-репозиторий>
    cd Pdf-extract-kit_VS_Docling
    ```

2.  **Создайте и активируйте виртуальное окружение:**
    ```bash
    # Windows
    python -m venv .venv
    .venv\Scripts\activate

    # Linux / macOS
    python3 -m venv .venv
    source .venv/bin/activate
    ```

3.  **Установите зависимости:**
    ```bash
    pip install -r requirements.txt
    ```

4.  **Загрузите модели (для MinerU):**
    Запустите специальный скрипт, который автоматически скачает и настроит необходимые веса моделей:
    ```bash
    python scripts/download_models.py
    ```

## Использование

1.  **Подготовка данных:**
    Скопируйте ваши PDF-файлы, которые нужно конвертировать, в папку:
    ```
    data/raw/
    ```

2.  **Запуск конвертации:**

    *   **Сравнение (запуск обоих методов):**
        Это режим по умолчанию. Он создаст две версии Markdown для каждого файла.
        ```bash
        python main.py
        ```

    *   **Только MinerU:**
        ```bash
        python main.py --mode mineru
        ```

    *   **Только Docling:**
        ```bash
        python main.py --mode docling
        ```

3.  **Получение результатов:**
    Готовые файлы ищите в папке `data/processed/`:
    *   `ИмяФайла_mineru.md` — результат работы MinerU.
    *   `ИмяФайла_docling.md` — результат работы Docling.
    *   `images/` — папка со всеми извлеченными изображениями.

## Структура проекта

```text
Pdf-extract-kit_VS_Docling/
├── main.py                 # Главная точка входа в программу
├── mineru.json             # Конфигурация моделей MinerU
├── requirements.txt        # Список зависимостей Python
├── src/
│   ├── mineru_converter/   # Адаптер для запуска MinerU
│   ├── docling_converter/  # Адаптер для запуска Docling
│   └── MinerU/             # Ядро библиотеки MinerU (интегрированное)
├── data/
│   ├── raw/                # Сюда класть PDF файлы
│   └── processed/          # Отсюда забирать MD файлы и картинки
├── models/                 # Папка с весами моделей (создается скриптом)
└── scripts/
    └── download_models.py  # Скрипт загрузки моделей
```

## Примечания

*   **Первый запуск**: При первом запуске MinerU может потребоваться время на инициализацию моделей. Последующие запуски будут быстрее.
*   **Ресурсы**: MinerU использует тяжелые нейросетевые модели. Если у вас мало оперативной памяти, попробуйте обрабатывать файлы по одному.
